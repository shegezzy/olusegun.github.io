about

Hi, I’m Olusegun a self taught, design thinking full-stack developer with a passion for motion design. I'm currently living in Lagos, Nigeria. I'm an absolute geek and love geeking out on all things. First fell in love with web design 7 years ago. I've done remote work for agencies, consukted for startups, and collaborated with talented people to create digital products for both business and consumer use. I'm quietly confident, naturally curious and perpetually working inproving my chops one design problem at time.

My favourite development stack at the moment is React. PROJECT TIMELINE

IT JUST DOESN’T HAPPEN IN A TWINKLE OF AN EYE!!!

PLANNING

Meeting the client ( Discussing the scope of the project)

1

DESIGN

Branding, Infrastructure & Layout In this phase, we set up the foundation of your project. 
Examples of project activities in this phase include:

  • The project title and tagline.
  • Logo, color palette, fonts, and page layouts.
  • A sitemap that shows all the projects pages and the relations between them.
  • Purchase a domain and hosting services.

    2

    • CONTENT PREPARATION

    The goal of this phase is to gather all the content needed for the project. The sitemap and the page layout styles created in the previous phase help identify the necessary content. Some content that are already available with the client. However, I might have to to work with external content creators in some cases. 
    Typical project activities in this phase include:

  • Identify the types of content needed. For example, page content, testimonials, privacy policy, terms of use, FAQs, etc.
  • Decide on client-side, internal, and external content creators and providers.
  • Create/update/receive the text and graphical content.
  • Organize the content in a content repository.
  • Proofread, validate and finalize content

    • 3

    development

    This is the phase where all the real work happens. Activities in this phase depend on the sitemap, layouts, and content collected in the previous three phases.
    Here are some examples of activities that happen during this phase.

  • Page elements such as buttons, call to actions, testimonials, etc.
  • Design pages based on layout styles and content.
  • Set up a sandbox server.
  • HTML, CSS, and Javascript validation.
  • Develop functionalities like a blog, an e-commerce store, or a CMS.
  • Organize and link pages according to the sitemap.
  • Review project with clients and get necessary approvals.

    • 4

    REVIEW

    In this phase, you validate the website’s functionality and verify that it matches the client’s requirements. Depending on the needs, various types of testing may be necessary. These can range from accessibility testing to website load testing. In some cases, I’ll also need to test the project for SEO-related issues.
    Examples of activities in this phase include:

  • Check that the website confirms with web standards.
  • Ensure accessibility standards for differently-abled persons.
  • Test the functionality is working as expected.
  • Check if the website is responsive and works well on all devices.
  • Fix issues that crop up during testing.

    • 5

    LAUNCH- Go Live

    Once you’ve tested the website and fixed the issues, it’s time to deploy the website on the client’s hosting server. Depending on the approved project proposal,
    I'll need to perform the following activities:

  • Upload the website to the client’s hosting server.
  • Write and hand over the project’s documentation to the client.
  • Train the client team to create and update the website.
  • Create a robots.txt file.
  • Create and submit the XML sitemap to search engines.
